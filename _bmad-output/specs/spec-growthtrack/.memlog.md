- (note) Input is two conflicting SRS drafts: v1 (WhatsApp reporting plus admin portal, Twilio POC, JWT) and v2 (adds AI forecasting, Redis and Celery, OAuth2). User directed reconciliation via bmad-help before this run.
- (decision) AI-based sales forecasting (v2 scope: LightGBM/XGBoost/Prophet forecast engine, Forecast entity, forecast widgets, territory/brand/doctor scoring, low-sales alerts) is Phase 2, not Phase 1, per explicit user directive at bmad-spec invocation.
- (capability) CAP-1 JWT authentication. intent: administrator logs in and holds a secure session via JWT. success: unauthenticated requests are rejected and a valid session expires or invalidates per policy.
- (capability) CAP-2 Dashboard. intent: administrator views todays sales, YTD, MTD, Achievement pct, Growth pct, team performance, and notification status in one place. success: all seven fields render from current data within the 3-second load constraint.
- (capability) CAP-3 Automated WhatsApp reporting. intent: system generates and sends a formatted daily performance report to configured recipients automatically, retrying failures and logging delivery status. success: each recipient receives exactly one correctly formatted report within 5 minutes of scheduled run, or a logged failure with retry attempts.
- (capability) CAP-4 Manual notification. intent: administrator selects recipients, composes a message, optionally attaches a report, and sends immediately. success: the message reaches selected recipients without waiting for the scheduled run and appears in history.
- (capability) CAP-5 Recipient management. intent: administrator manages individual users, WhatsApp groups, WhatsApp channels, and sales teams used to target notifications. success: adding, editing, or removing a recipient or group changes who future notifications reach.
- (capability) CAP-6 Brand analytics. intent: system surfaces top-selling, low-performing, and recommended focus brands from current sales data. success: all three lists compute from a given dataset and appear in both dashboard and daily report.
- (capability) CAP-7 Doctor target list. intent: system surfaces a prioritized doctor visit list per territory with name, territory, and target priority. success: each report includes a doctor list ranked by target priority for the relevant territory.
- (capability) CAP-8 Notification history. intent: administrator views a full history of sent notifications with date, time, recipient, message type, and delivery status. success: every notification the system sends, scheduled or manual, appears in history with accurate status the same operational day.
- (constraint) Dashboard must load within 3 seconds.
- (constraint) Automated notification generation must complete within 60 seconds.
- (constraint) System must support 500 or more concurrent users.
- (constraint) Scheduled notifications must be delivered within 5 minutes of scheduled execution time.
- (constraint) All communication must be over HTTPS.
- (constraint) Passwords must be stored encrypted, never plaintext.
- (constraint) Access control must be role-based across Administrator, Sales User, and Manager roles.
- (constraint) All administrative actions must be audit-logged.
- (constraint) System must maintain 99.5 percent uptime with automatic recovery after failures.
- (constraint) No duplicate notifications may be sent for the same send event; failed sends must auto-retry.
- (non-goal) AI-based sales forecasting is out of scope for this phase; full scope preserved in roadmap-phase2.md companion.
- (non-goal) Interactive charting dashboard, image-rich WhatsApp messages, Power BI integration, email notifications, push notifications, native mobile app, scheduled report customization, multi-language support, and PDF or Excel export are out of scope; source concept doc lists these as future enhancements.
- (non-goal) Production-grade WhatsApp Business Platform migration is out of scope; Phase 1 uses Twilio as a POC-only provider.
- (decision) Success signal: a recipient receives an accurate, correctly formatted WhatsApp report within 5 minutes of the scheduled daily run with zero duplicate sends, and an administrator can independently trigger, retarget, and audit any notification, all without touching source data systems directly.
- (assumption) Domain is pharmaceutical or FMCG field sales; doctor targeting plus brand performance implies a medical or pharma detailing use case per v2 framing.
- (assumption) Deployment locale is likely Bangladesh; sample report uses Cr BDT (Bangladeshi crore taka) currency formatting.
- (assumption) Doctor entries refer to healthcare professionals being visited by sales reps, not patients; no patient health data is implicated by the entities described.
- (question) What system is the source of truth for sales data: ERP, CRM, or another database?
- (question) What time should the daily report run, and can individual recipients customize their schedule?
- (question) Are users created manually, or synced from an existing system such as Active Directory or an HRIS?
- (question) How exactly are users mapped to WhatsApp groups, channels, or individuals for targeting?
- (question) What system or systems supply brand and doctor data, and how often are they refreshed?
- (question) Do managers get direct web portal access, or only WhatsApp-delivered reports? v1 grants managers view team performance but FR-1 names only Administrator as a portal login role.
- (question) What are the exact formulas for Sales Achievement pct, YoY Growth pct, and MoM Growth pct?
- (question) Which WhatsApp provider is intended for production: stay on Twilio, or migrate to Meta WhatsApp Business Platform? v1 flags Twilio as POC-only and v2 lists both without deciding.
- (question) Is OAuth2 required for Phase 1 authentication, or is JWT alone sufficient until Phase 2? v2 adds OAuth2 without stated reason.
- (question) Is a Redis and Celery background-job layer needed in Phase 1, for example for retry queues or scheduling, or is it introduced specifically for Phase 2 forecasting workloads?
- (question) Are pharmaceutical marketing or promotional-compliance rules a concern for how doctors are targeted or prioritized? Neither source document addresses this.
